Slidecode

A slide component helps layout the UI in a systematic and organized way. This component enables the UI to incrementaly hide or show components using a recursive sliding technique, in turn enabling the ability to methodologically design interfaces which allow users to quickly navigate through complex workflow patterns. The principle is simple, a wrapper component with some css and javascript for a flex-box based div that can be nested within itself while also being slideable, e.g for revealing and hiding different parts of the ui. A slide can be split into any amounts of "slides", each slides area is controlled by the parent and betaalong with manual constraints.

Application components with their respective css are nested inside each slide, the slides themselves should not be touched with css or any exterior javascript.

Hover over some of the slide components on the right to see the effect. Each one of those elements is a slide component, either as a parent for nested one of a child. each child contains its own card element.

require "intui/Slide"
h Slide,
	vert: yes #make it vertical!
	slide: yes #
	h Slide,
		class: 'my-class'

props
fixedboolfixed grids are not scrollable. When there is no more space left, any new items added to array will replace oldest dispalyed items, this type of grid is handy for displaying live feeds, although you can also unshift items in scrollable grids.
vertverticalboolwhen a grid is not fixed, we can set the scroll direction
classNamestringthe grid container class
sizeintAn integer representing how many units the grid is across. set this based on how you want to organize your items and to what resolution. keep in mind that items with more variant unit height and widths can create empty spaces. Higher size also leads to longer processing times. sometimes its good idea to set size as a multiple of 2 for both grid and items.
lengthintAn integer representing how many units the grid is in length. This variable is ignored for non fixed grids!
bufferOffsetUnitsintThe buffer units from start of visible grid items to end, the only grid items that are rendered are the ones that are visible in the container. This property manages how far from the edges of the container the units get rendered.
animationOffsetUnitsBetafloatrendered grid items are not nessesarily visible. Their visibility and animations are managed with this prop. beta is the % of individual unit size. 1.5 = 150% of 1 unit.
freeSpotfunctionFor fixed grids, sometimes you want to free up units removing stale items in a specific pattern that matches your business logic, use this function to override the default freeSpot method
onUnitMouseEnterfunctionWhen mouse enters a unit, unit r and c is passed to this function.
